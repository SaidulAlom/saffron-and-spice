# Portfolio Demo Flow

Use this sequence for a clean 3-5 minute walkthrough.

## 1. Landing impression

- Open the homepage in dark mode.
- Let the loader finish and scroll once to show reveal motion and parallax.
- Point out the refined hero, hover polish, and responsive CTA layout.

## 2. Brand storytelling

- Visit `About` and `Experiences`.
- Mention that the project is not just a static restaurant skin; it uses narrative pages to make the brand feel premium.
- On mobile, show that spacing, headings, and card stacks stay readable.

## 3. Commerce journey

- Open `Menu`.
- Filter categories.
- Add 2-3 items to the cart.
- Open the cart sidebar and explain the interactive state changes.

## 4. Real form flows

- Open the reservation modal and mention backend-connected submissions.
- Visit `Contact` and show loading, validation, and success state behavior.
- Mention CSRF-protected write routes and structured API responses.

## 5. Reliability story

- Explain that read-side Supabase content has fallback data.
- Show `Testimonials` and `Gallery` to emphasize that the portfolio demo remains stable even if content fetches fail.

## 6. Production polish

- Mention route-level SEO metadata, sitemap generation, manifest, OG image, and Railway deployment config.
- Close by referencing the full-stack setup: React, TypeScript, Supabase, and Express.

## Optional talking points

- Theme preference is persisted locally.
- The backend health route can be smoke-tested quickly.
- The project is intentionally admin-free because it is optimized for portfolio presentation, not internal restaurant operations.
